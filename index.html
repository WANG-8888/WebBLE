Firmware update over Bluetooth for SMP-server devices (Zephyr / nRF Connect SDK). Speaks the same protocol as nRF Device Manager.

00 Project

Each project remembers its device-name filter, write size, image slot, and its own parameter commands. Saved on this device; use Export to back up or move projects to another phone.

01 Connect

Requires Chrome / Edge / Opera (desktop or Android), served over HTTPS, localhost, or opened directly as a local file. iOS Safari and Firefox have no Web Bluetooth.

02 Transport

Web Bluetooth doesn't expose the negotiated MTU, so this is set manually. 244 is safe for the common 247-byte MTU. Lower it if writes fail; raise it only if your firmware has SMP packet reassembly enabled.

04 Upload firmware

0%—0 / 0 B

Use the MCUboot-signed binary your build produces (typically app_update.bin / *.signed.bin), not the raw .hex or unsigned image.

Recommended flow: Upload → Test & reset. The device swaps and boots the new image as a trial; if it isn't confirmed it reverts on the next reset. After it reboots and you reconnect, press Confirm to keep it permanently. On Matter-over-Thread devices, BLE advertising often stops after commissioning — trigger your SMP advertising (e.g. the button you use for nRF Device Manager) before reconnecting.

05 Parameters · custom SMP

Add / edit command

Application parameters live in a custom SMP group (IDs 64+) you define in firmware. Save the ones you use per project for one-tap access above. JSON supports numbers, strings, booleans, arrays and nested objects; byte fields aren't expressible here — send me a group's schema and I'll wire typed inputs for it.
